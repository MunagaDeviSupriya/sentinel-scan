## Overview

MCOGAN+ detects whether a file is **Benign** or **Malicious** by analysing five static PE features. When a file is flagged as malicious, the system further enriches the verdict with a descriptive category (e.g. Ransomware, Trojan, Dropper) using feature heuristics. A live file monitor watches configured folders and pushes results to an interactive web dashboard in real time via Server-Sent Events (SSE).

## Features

- **Static malware classification** using five PE-header features
- **Malware sub-categorisation** вЂ” Ransomware, Trojan, Dropper, Worm, Spyware, Packed/Obfuscated Malware
- **Real-time live monitor** вЂ” watches folders via `watchdog`, auto-scans downloaded files
- **Server-Sent Events (SSE)** вЂ” pushes scan results to the dashboard without polling
- **Feature importance** вЂ” XGBoost gain scores returned with every prediction
- **Optional SHAP explanations** вЂ” enable by setting `USE_SHAP = True` in `backend.py`
- **MITRE ATT&CK mapping** вЂ” dashboard maps known malware families to ATT&CK techniques
- **REST API** вЂ” simple `/predict` endpoint accepts a JSON feature vector

## Setup & Installation

### Prerequisites

- Python 3.8+
- pip

### 1. Clone the repository

```bash
git clone https://github.com/<your-username>/mcogan-plus.git
cd mcogan-plus
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Train the model (skip if `model.pkl` is already present)

```bash
python train_model.py
# Options:
#   --data path/to/dataset.csv   (default: dataset_malwares.csv)
#   --estimators 300             (XGBoost trees)
#   --max-depth 6
#   --lr 0.05
#   --cv-folds 5
```

This produces `model.pkl` and `label_encoder.pkl`.

## Usage

### Start the Flask backend

```bash
python backend.py
# Runs on http://127.0.0.1:5000
```

### Open the dashboard

Open `index.html` in your browser (no web server needed вЂ” it connects to the local API).

### Start the live file monitor

```bash
# Watch default ~/Downloads folder
python monitor.py

# Watch a custom folder
python monitor.py --folder "C:/Users/YourName/Downloads"

# Watch multiple folders recursively
python monitor.py --folder "C:/A" "C:/B" --recursive

# Use polling observer (for USB / network drives)
python monitor.py --polling
```

The monitor will detect new files, extract features, call the backend, print results to the terminal, and push them to the dashboard in real time.

## API Reference

### `GET /health`

Returns backend status and number of connected SSE clients.

```json
{ "status": "ok", "sse_clients": 1 }
```

### `POST /predict`

Classify a file given its feature vector.

**Request body:**
```json
{
  "features": [204800, 6.82, 4, 12, 85]
}
```

Feature order: `[File Size, Entropy Score, Section Count, API Call Count, Import Table Size]`

**Response:**
```json
{
  "prediction":    "Ransomware",
  "confidence":    91.4,
  "is_malicious":  true,
  "probabilities": [{ "family": "Malware", "pct": 91.4 }, { "family": "Benign", "pct": 8.6 }],
  "shap":          [{ "name": "Entropy Score", "value": 0.312, "pct": 44.1 }, ...]
}
```

### `GET /events`

SSE stream вЂ” subscribe to receive real-time scan results pushed by the monitor.

### `POST /monitor_event`

Used internally by `monitor.py` to forward scan results to the SSE broadcast channel.

## Dataset

`dataset_malwares.csv` contains PE-header features extracted from benign and malicious Windows executables. Key columns used for training:

| CSV Column                   | Model Feature      |
|------------------------------|--------------------|
| `SizeOfImage`                | File Size          |
| `SectionMaxEntropy`          | Entropy Score      |
| `NumberOfSections`           | Section Count      |
| `SuspiciousImportFunctions`  | API Call Count     |
| `DirectoryEntryImportSize`   | Import Table Size  |
| `Malware`                    | Label (0/1)        |
